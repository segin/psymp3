# PsyMP3 Codebase Audit — 2026-04-12

## Executive Summary

PsyMP3 is a well-structured C++17 audio player with clear architecture, comprehensive
documentation, good build-system hygiene (security hardening flags, sanitizer support,
`-Wall -Werror`), and a large test suite (383 test files). The build compiles cleanly
with zero warnings. The main areas of concern are **input validation on untrusted
media/network data**, **MPRIS and streaming-thread correctness**, and **test-suite
build drift that currently blocks `make check`**.

## 2. Security: Input Validation (Media Parsing)

These are the highest-risk findings. A malicious audio file is the primary attack vector.

| Severity | Issue | Location |
|----------|-------|----------|
| **HIGH** | **FLAC SampleReconstructor**: `upscaleTo16()` computes `16 - source_bit_depth` as unsigned; values 13–15 fall through the default case and underflow is possible for values > 16. | `src/codecs/flac/SampleReconstructor.cpp` |
| **HIGH** | **FLAC ResidualDecoder**: `1u << partition_order` with no validation that `partition_order < 32`. | `src/codecs/flac/ResidualDecoder.cpp` |
| **HIGH** | **FLAC MetadataParser**: A parsed `sample_rate` of 0 from STREAMINFO causes divide-by-zero downstream. | `src/codecs/flac/MetadataParser.cpp` |
| **HIGH** | **PCM Codec**: `convertSamples()` assumes input size is evenly divisible by `bytes_per_sample`; remainder bytes can cause an over-read. | `src/codecs/pcm/PCMCodecs.cpp` |
| **MEDIUM** | **ISO SampleTableManager**: `sampleCount * sizeof(uint32_t)` can overflow before allocation, enabling DoS via huge sample counts. | `src/demuxer/iso/SampleTableManager.cpp` |
| **MEDIUM** | **Ogg Seeking**: Offset arithmetic in `OggSyncManager` has potential integer overflow in seek calculations. | `src/demuxer/ogg/OggSyncManager.cpp` |
| **MEDIUM** | **XML Parser**: No recursion depth limit in `parseElement()` — deeply nested XML causes stack overflow. | `src/core/utility/XMLUtil.cpp` |

## 3. Security: Network & Credential Handling

| Severity | Issue | Location |
|----------|-------|----------|
| **MEDIUM** | **HTTP Content-Length**: Parsed with `std::stol()` but no upper-bound validation; negative or excessively large values are accepted. | `src/io/http/HTTPIOHandler.cpp` ~L97 |
| **MEDIUM** | **M3U Path Traversal**: `joinPaths()` concatenates playlist-relative paths without sanitizing `..` sequences; a crafted `.m3u` file can reference arbitrary filesystem paths. | `src/playlist.cpp` ~L408 |
| **MEDIUM** | **Scrobble Cache Deserialization**: `std::stoi()`/`std::stoll()` calls without try/catch; a corrupted cache file crashes the scrobbler. | `src/lastfm/scrobble.cpp` |
| **LOW** | **URI Scheme**: No allowlist validation; arbitrary schemes accepted beyond `file://` and `http(s)://`. | `src/io/URI.cpp` |

**Positive note:** Last.fm credentials are properly handled with `umask(0077)`,
`OPENSSL_cleanse()`, and legacy plaintext-password migration.

## 4. Threading & Concurrency

| Severity | Issue | Location |
|----------|-------|----------|
| **CRITICAL** | **MPRIS `Seeked` signal deadlock**: `emitSeeked_unlocked()` queues a lambda that locks `m_mutex`, but the worker invokes queued signal functions while already holding `m_mutex`. The first queued `Seeked` signal can wedge the signal worker. Normal seek paths reach this through `Player::seekToInternal()` → `MPRISManager::notifySeeked()`. | `src/mpris/SignalEmitter.cpp`, `src/player.cpp`, `src/mpris/MPRISManager.cpp` |
| **CRITICAL** | **SignalEmitter shutdown use-after-free risk**: the destructor calls `stop(false)`, which can detach a worker thread that still captures `this` and accesses member state after object destruction. `stop_unlocked()` also manually unlocks/relocks a mutex owned by an outer `std::lock_guard`, violating RAII ownership assumptions. | `src/mpris/SignalEmitter.cpp` |
| **HIGH** | **StreamingManager demuxer race**: `readChunk()` directly calls `m_demuxer->readChunk()` without taking `m_mutex`, while the background streaming thread and `seekTo()` access the same demuxer under the mutex. Direct reads can interleave with background reads/seeks. | `src/io/StreamingManager.cpp`, `include/io/StreamingManager.h` |
| **MEDIUM** | **Audio callback raw-pointer access**: `m_current_stream_raw_ptr` (atomic) can become a dangling pointer if `setStream()` replaces `m_owned_stream` concurrently. The `shared_ptr` copy in the callback mitigates this partially, but the TOCTOU gap remains. | `src/audio.cpp` |
| **MEDIUM** | **Player `updateState()`**: Reads `audio->getCurrentStream()` as a raw pointer without a lock that prevents the stream from being deleted between checks. | `src/player.cpp` |
| **LOW** | **HTTPIOHandler::initialize()**: Double-check locking pattern checks `m_initialized.load()` before acquiring `m_initialization_mutex` (TOCTOU). | `src/io/http/HTTPIOHandler.cpp` |

**Positive note:** The public/private lock pattern (`_unlocked` methods) is used
throughout the MPRIS, widget, and I/O subsystems, but `SignalEmitter` and
`StreamingManager` need correction where callbacks or demuxer operations escape the
intended locking model.

## 7. Recommendations (Priority Order)

1. **Fix `SignalEmitter` locking and shutdown**: Execute queued signal callbacks
   outside `m_mutex`, replace manual unlock/relock with `std::unique_lock`, and make
   the destructor join the worker thread instead of detaching it.
2. **Fix `make check`**: Correct `test_widget_hierarchy_properties.cpp` to use public
   methods `getParent()` and `getChildren()` instead of accessing protected members.
3. **Serialize `StreamingManager` demuxer access**: Ensure direct reads, background
   reads, and seeks all use one synchronization policy.
4. **Clean up test makefile drift**: Remove duplicate `test_extensibility_utils`
   definitions and either add or delete the orphaned `test_widget_hierarchy_properties`
   target variables.
5. **Add bounds validation to FLAC parser**: Validate `partition_order`, `sample_rate`,
   and `bit_depth` from STREAMINFO before use.
6. **Guard PCM conversion**: Check `input_size % bytes_per_sample == 0` before
   processing.
7. **Validate HTTP Content-Length**: Reject negative values and cap at a reasonable
   maximum (e.g., 10 GB).
8. **Sanitize M3U relative paths**: Reject or normalize paths containing `..`
   components.
9. **Implement `MediaFile::exists()`**: Use `std::filesystem::exists()` instead of
   returning `true`.
10. **Document the `seek_error` debug channel** in the help/about debug channel list.
11. **Add recursion depth limit** to XML parser.
12. **Add fuzz-testing targets** for demuxer and codec entry points.
13. **Fix duplicate `AC_DEFINE`** for `ENABLE_ALAW_CODEC` in `configure.ac`.
